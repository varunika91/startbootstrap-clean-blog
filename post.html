At Xilinx

OpenCL runtime QoR

OpenCL runtime QoR for Xilinx SDAccel framework for running nightly tests to identify bottlenecks in the SDAccel tool design. Tests were run on Xilinx ADM PCIe ku3 and 7v3 boards.
OpenCL APIs tested:
- Buffer object (Buffer creation, deletion, migration, map)
- Compute test cases (bitstream download, CU start time)
- Memory usage (getting platform and device ID, context and command queue creation)
- Multi-threading support (creating multiple threads in host code each using a spearate command queue)
Created comparison statistics of the performance of various OpenCL APIs with SDAccel v2015.4 and SDAccel v 2016.1

Kernel Hacking

Kernel hacking of TCP module of network stack of Linux kernel to remove the exponential back-off and modify slow start phase. Achieved 12 Mbps over 200ms delay and 25% packet loss. Used TCP Hybla for improving throughput.

Custom File Transfer Protocol

Implemented a UDP based reliable transfer protocol in C, achieving 50Mbps over a high-delay link of 200ms and loss rate of 20%. Used sequencing and NACK to ensure reliability

Software Defined Network Implementation
